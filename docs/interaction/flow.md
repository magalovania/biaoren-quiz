# 镖人角色测试 - 交互流程设计

## 1. Primary Persona

**林峰，22岁，大学生，漫画爱好者**

- 看过《镖人》动画，对角色有基本了解
- 喜欢在朋友圈分享测试结果
- 手机重度用户，注意力有限
- 目标：快速完成测试，获得有趣可分享的结果

---

## 2. 完整用户流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  首页   │ →  │  测试   │ →  │  计算   │ →  │  结果   │ →  │  分享   │
│  入口   │    │  答题   │    │  匹配   │    │  展示   │    │  传播   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
    3s            60-90s          2s            15s            持续
```

### 2.1 首页入口

**状态**：初次访问
**用户目标**：了解这是什么、决定是否参与
**界面元素**：
- 大标题：「测一测你是镖人中的哪个角色」
- 副标题：简短描述（15道题，1分钟）
- 主按钮：「开始测试」
- 底部：已有 X 人完成测试（社交证明）

**关键交互**：
- 点击「开始测试」→ 进入测试页，带淡入动效

### 2.2 测试答题

**状态**：答题中
**用户目标**：完成所有题目
**界面元素**：
- 顶部：进度条 + 题号（如 3/15）
- 中部：问题文字（武侠情境化）
- 下部：3-4个选项按钮
- 底部：留白呼吸

**关键交互**：
- 点击选项 → 选项高亮 → 300ms后自动进入下一题
- 最后一题答完 → 显示加载动画 → 进入结果页

**防止流失**：
- 题目控制在15题以内
- 每题3-4选项，避免决策困难
- 无需返回修改（简化流程）

### 2.3 计算匹配

**状态**：计算中
**用户目标**：等待结果
**界面元素**：
- 加载动画（角色剪影逐渐清晰）
- 提示文字：「正在匹配你的灵魂角色...」

**时长**：1.5-2秒（营造期待感）

### 2.4 结果展示

**状态**：查看结果
**用户目标**：了解匹配角色、获得情感满足
**界面元素**：
- 角色立绘/剪影
- 角色名称（大字）
- 匹配度百分比
- 性格描述（3-5条）
- 金句/台词
- 五维属性雷达图
- 分享按钮

**关键交互**：
- 结果卡片有揭示动效
- 雷达图动态绘制
- 点击「分享」→ 展示分享选项

### 2.5 分享传播

**状态**：分享
**用户目标**：分享到社交平台
**分享方式**：
- 生成海报图片（微信长按保存）
- 复制链接
- 分享到微博（带文案）

## 3. 测试题目设计

### 3.1 题目类型

采用**情境代入题**，将用户置于武侠世界中：

| 类型 | 示例 | 数量 |
|------|------|------|
| 价值观选择 | "在江湖中，你最看重什么？" | 4题 |
| 行为反应 | "同伴被强敌围困，你会？" | 4题 |
| 性格倾向 | "面对不公，你通常？" | 4题 |
| 人际关系 | "你更愿意与怎样的人结伴？" | 3题 |

**总计：15题**

### 3.2 维度映射

基于**五因素人格模型**映射到武侠维度：

| 人格维度 | 武侠维度 | 对应角色特质 |
|----------|----------|--------------|
| 开放性 | 自由度 | 刀马(高) vs 常贵人(低) |
| 尽责性 | 信义值 | 刀马(高) vs 竖(低) |
| 外向性 | 社交力 | 燕子娘(高) vs 刀马(低) |
| 宜人性 | 仁心值 | 阿育亚(高) vs 常贵人(低) |
| 神经质 | 情绪性 | 竖(高) vs 刀马(低) |

### 3.3 题目示例

```json
{
  "id": 1,
  "question": "你独自走在荒漠中，前方出现一群可疑之人，你会？",
  "options": [
    { "text": "拔刀相向，先下手为强", "weights": { "武力": 3, "情绪": -1 } },
    { "text": "冷静观察，判断对方意图", "weights": { "智谋": 3, "情绪": 1 } },
    { "text": "主动搭话，探听消息", "weights": { "社交": 3, "自由": 1 } },
    { "text": "绕道而行，避免麻烦", "weights": { "仁心": 2, "信义": -1 } }
  ]
}
```

### 3.4 匹配算法

```
用户得分 = Σ(各维度分数)
角色距离 = √Σ((用户维度 - 角色维度)²)
匹配角色 = 角色距离最小者
匹配度 = (1 - 距离/最大距离) × 100%
```

### 4.2 分享卡片规范

**尺寸**：750 × 1334 px（微信标准）

**构成**：
- 背景：渐变 + 水墨纹理
- 角色立绘：居中或偏上
- 文字：角色名 + 金句
- 品牌水印：右下角
- 二维码：底部

## 5. 关键交互点

### 5.1 题目切换

**问题**：用户可能感到进度缓慢
**方案**：
- 进度条实时更新
- 题号醒目显示
- 平滑过渡动效

### 5.2 结果揭示

**问题**：期待感如何营造
**方案**：
- 加载时显示剪影
- 2秒后剪影渐变清晰
- 角色名伴随震动效果出现

### 5.3 分享引导

**问题**：用户可能忽略分享
**方案**：
- 结果展示后自动弹出分享提示
- 高匹配度(>85%)显示"太准了"文案
- 分享按钮始终可见

### 5.4 错误处理

| 场景 | 处理 |
|------|------|
| 网络中断 | 本地存储答案，恢复后继续 |
| 数据加载失败 | 显示友好提示，提供重试 |
| 分享失败 | 提供多种分享方式备选 |
