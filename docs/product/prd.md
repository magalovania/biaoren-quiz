# 《镖人》角色测试 产品需求文档

> 测一测你是镖人中的哪个角色

---

## 一、目标用户画像

### 1. 核心粉丝型
- **年龄**：18-35岁
- **特征**：《镖人》漫画/动画核心受众，对角色有深度认知
- **动机**：验证自我认知与角色契合度，寻求情感投射与认同
- **行为**：主动搜索测试，完成后高度分享意愿，会讨论结果准确性
- **痛点**：市面测试流于表面，缺乏对角色内核的深度挖掘

### 2. 社交娱乐型
- **年龄**：16-28岁
- **特征**：泛二次元用户，对《镖人》有基础了解或被分享触达
- **动机**：社交货币获取，参与朋友圈话题讨论
- **行为**：测试速度快，重视分享效果，关注结果是否"有梗"
- **痛点**：测试流程过长会流失，结果不够有传播性

### 3. 新用户探索型
- **年龄**：20-40岁
- **特征**：对武侠文化感兴趣，但未接触《镖人》
- **动机**：好奇心驱动，通过测试了解作品
- **行为**：测试中会仔细阅读角色介绍，可能转化为新读者
- **痛点**：测试与原著割裂，无法激发进一步了解的兴趣

## 三、用户体验策略

### 3.1 测试流程设计（认知负荷最小化）

```
入口 → 简短引导(3s) → 测试(60-90s) → 结果生成(2s动画) → 结果展示 → 分享引导
```

**设计原则**：
- **渐进式披露**：每题只展示必要信息，避免认知过载
- **即时反馈**：选择后轻微动效确认，维持心流状态
- **预期管理**：进度条清晰，剩余题数可见

### 3.2 题目设计策略

基于五因素人格模型（大五人格）设计题目维度：

| 人格维度 | 对应角色特质 | 题目数量 |
|---------|-------------|---------|
| 开放性 | 刀马的"自由"vs常贵人的"秩序" | 2-3题 |
| 尽责性 | 刀马的"使命感"vs竖的"随性" | 2-3题 |
| 外向性 | 燕子娘的"社交力"vs刀马的"孤僻" | 2-3题 |
| 宜人性 | 阿育亚的"善良"vs常贵人的"冷酷" | 2-3题 |
| 神经质 | 竖的"情绪波动"vs刀马的"冷静" | 2-3题 |

**题目风格**：
- 使用武侠情境而非抽象问题
- "如果你在江湖中..."情境代入
- 每题3-4选项，避免决策困难

### 3.3 结果展示策略

**情感峰值设计**：
1. **悬念营造**：加载动画展示剪影逐渐清晰
2. **角色揭晓**：角色原画+名称震撼呈现
3. **性格解读**：3-5条精准描述，用户可自我验证
4. **匹配金句**：一句有传播力的总结（"江湖路远，刀马心间"）
5. **扩展内容**：角色介绍、作品引导

### 3.4 情感设计三层级

- **本能层**：水墨风视觉、硬派武侠美学、动效流畅
- **行为层**：测试流畅、结果准确、分享便捷
- **反思层**：结果引发自我思考、激发对作品的兴趣

## 四、分享机制设计

### 4.1 分享卡片设计

**微信/微博分享卡片**：
- 尺寸：500x400px（微信）或 800x600px（微博）
- 构成：角色原画 + 匹配度 + 金句 + 品牌水印
- 风格：水墨质感，边框装饰，印章落款

**文案模板**：
- 主标题：「测出你是【刀马】」
- 副标题：「江湖险恶，唯有快刀斩乱麻」
- 底部：「扫码测测你是谁」

### 4.2 分享触发点

| 触发点 | 时机 | 引导方式 |
|-------|------|---------|
| 结果页底部 | 查看完结果后 | "分享给好友测测"按钮 |
| 结果页顶部 | 匹配度>80%时 | "太准了！分享炫耀"浮层 |
| 角色介绍后 | 阅读完角色资料 | "找到同角色好友"CTA |

### 4.3 病毒系数优化

- **对比触发**：「你的好友中，只有12%是刀马」
- **挑战触发**：「敢不敢测测你是不是真正的刀马？」
- **社交证明**：「已有X人完成测试」

## 五、成功指标

### 5.1 核心指标（北极星）

**分享率** = 分享用户数 / 完成测试用户数
- 目标：≥35%

### 5.2 过程指标

| 指标 | 定义 | 目标值 |
|-----|------|-------|
| 开始率 | 点击开始/页面访问 | ≥60% |
| 完成率 | 完成测试/开始测试 | ≥80% |
| 平均测试时长 | 完成测试总时长/完成人数 | 60-90秒 |
| 分享点击率 | 点击分享/完成测试 | ≥50% |
| 分享回流率 | 通过分享链接进入/分享点击 | ≥15% |

### 5.3 业务指标

| 指标 | 定义 | 目标值 |
|-----|------|-------|
| 新用户占比 | 首次访问/总访问 | ≥70% |
| 原著转化率 | 点击了解原著/完成测试 | ≥10% |
| 二次分享率 | 回流用户分享/回流用户 | ≥20% |

### 5.4 体验质量指标

| 指标 | 定义 | 目标值 |
|-----|------|-------|
| 结果认同度 | "结果很准"点击率 | ≥60% |
| 页面加载时间 | 首屏加载 | <2秒 |
| 错误率 | 测试中断/开始测试 | <3% |

## 六、角色匹配逻辑

### 6.1 角色画像矩阵

```
        冷酷 ───────────── 温暖
          │
    刀马 ╳│          ╳ 阿育亚
          │
    竖   ╳│        ╳ 燕子娘
          │
   常贵人 ╳│
          │
        理性 ───────────── 感性
```

### 6.2 匹配算法

```
角色匹配度 = Σ(维度得分 × 维度权重) × 角色系数
```

每个角色预设维度基准值，用户得分与各角色基准值计算欧氏距离，取最近角色。

### 6.3 结果展示

- 主匹配角色：匹配度最高（80-100%显示）
- 次匹配角色：匹配度第二（60-80%显示）
- 性格维度雷达图：可视化五维度得分

## 七、技术要求
